    TP Simulation d'une machine & Assembleur

    Un simulateur basique

    On se propose de mettre en pratique les connaissance acquises grâce au simulateur développé par Peter L. Higgison. Ce simulateur est basé sur l'architecture de Von Neumann étudié précédemment, on y retrouve donc un processeur, ses registres et une mémoire.

    L'interface de ce simulateur est découpé en quatre grandes parties :

    • Le programme où vous allez saisir vos différents programmes en assembleur.

    • Le processeur avec l'état de ses différents registres (notamment l'adresse de la prochaine instruction PC, ses registres de travail R0 à R12, ses drapeaux N Z C V).

    • La mémoire dont vous pouvez modifier la représentation des valeurs (hexadécimale, binaire, décimal etc.)

    • Les entrées / sorties qui permet à la machine de communiquer avec le monde extérieur !

    Des valeurs de 32 bits ?

    Comme vous pouvez le constater, une cellule de la mémoire du simulateur correspond à 32 bits. La mémoire est pourtant bien découpée en octets, les adresses font bien référence à des octets. On ne pourra hélas utiliser que les adresses divisibles par 4 dans notre programme.

    1. Les adresses de la mémoire de cette machine vont de 0x0000à 0xFFFFF. Quelle est la taille de la mémoire ?

    2. Quelle est la taille en bits d'un registre du processeur dans cette simulation ?

    Un premier programme

    Copier ce programme (bouton EDIT) :

        MOV R2, #0xABCD ; (1)!
    MOV R7, #0x1234
    ADD R0, R2, R7 ; (2)!
    HALT ; (3)!

    1. R2 0xABCD Le symbole # permet de préciser une valeur immédiate. MOV vient du mot anglais move, déplacer.

    2. R0 R2 + R7

    3. Permet d'arrêter (ou mettre en pause) l'exécution du programme.

    Lorsque le programme est assemblé (bouton SUBMIT), le programme est stocké dans la mémoire, prêt à être exécuté :

    0xe30a2bcd 0xe3017234 0xe0820007 0xe1000070

    Adressage indirect

    Un registre R peut stocker une adresse (le registre est alors appelé pointeur). On écrit [R] pour récupérer la valeur à cette adresse dans la mémoire. Par exemple LDR R0, [R1] affecte à R0 la valeur à l'adresse R1 (et non la valeur de R1 en elle-même).

    Les labels

    Un label (ou étiquette en français) permet de nommer symboliquement une adresse. 

        MOV R0, #0
mon_label: ;(1)!
    ADD R0, R0, #1
    B mon_label ;(2)!
    HALT

    1. mon_label désigne ici l'adresse de l'instruction suivante ADD R0, R0, #1, c'est-à-dire 0x04.

    2. B est un saut (ou branchement) inconditionnel, il permet de sauter à une instruction particulière grâce à son adresse (mon_label).

    Saut relatif

    On peut sauter à un label, mais aussi un nombre d'instructions relativement à sa position. Par exemple : B .+3 sautera jusqu'à la 3ème prochaine instruction (en ignorant donc les 2 instructions entre). B .-5 sautera 5 instructions en arrière.

    Les conditions et les boucles

    Si les instructions if et while n'existent pas en assembleur, il existe à la place les sauts conditionnels. On compare deux valeurs à tester, et suivant le résultat de cette comparaison on saute ou non à une ligne de code (grâce à un label ou relativement).

    Comparaison

    On compare une valeur d'un registre avec une autre valeur grâce à l'instruction CMP (compare). Le résultat de cette comparaison est stocké dans les différents drapeaux (N Z C V) du processeur.

        MOV R0, #10
    CMP R0, #20 ;(1)!
    HALT
    1. Compare la valeur du registre R0 avec la valeur immédiate 20.

    Sauts conditionnels

    Une fois les drapeaux modifiés par CMP, on effectue un saut conditionnel suivant le résultat de la comparaison grâce aux instructions :

    Instruction Symbole Signification
    BEQ label \(=\) Branch if EQqual
    BNE label \(\ne\) Branch if Not EQqual
    BLT label \(<\) Branch if Less Than
    BGT label \(>\) Branch if Greater Than

    Modifions le code de la section précédente, en transformant le saut inconditionnel B en un saut conditionnel :

        MOV R0, #0
mon_label:
    ADD R0, R0, #1
    CMP R0, #10
    BLT mon_label
    HALT

    Hello, World!

    Raghu Venkatesh

    If you can write "hello world" you can change the world.

    Je n'allais tout de même pas vous apprendre l'assembleur sans vous faire afficher « Hello, World. » dans la console ! Pour afficher un message dans la console, il faut d'abord le stocker dans la mémoire :

    mon_message:
    .ASCIZ "Hello, World!"
    Une chaîne de caractères dans la mémoire

    Regardons la mémoire après avoir assemblé le code précédent :

    0x6c6c6548  0x57202c6f  0x646c726f  0x00000021

    Les caractères sont codés en ASCII donc sur un octet, on découpe ainsi :

    6c 6c 65 48    57 20 2c 6f    64 6c 72 6f    00 00 00 21

    On regarde enfin la table ASCII pour déterminer les caractères correspondants :

    l  l  e  H     W     ,  o     d  l  r  o           \0 !

    Pourquoi les caractères sont renversés ? La notion de boutisme.

    On laissera ces données à la fin du programme. On affecte la première adresse de ce message dans un registre, comme R0, puis on utilise STR pour l'envoyer à la console :

        MOV R0, #mon_message ;(1)!
    STR R0, .WriteString ;(2)!
    HALT

mon_message:
    .ASCIZ "Hello, World!"

    1. On copie l'adresse de début de mon_message, soit 0xC.

    2. On envoie l'adresse 0xC à la console qui va ensuite lire le contenu du message.

    2. Écrire le programme qui demande à l'utilisateur son âge et affiche s'il est majeur ou non.
